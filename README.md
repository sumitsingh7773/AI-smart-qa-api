Smart Q&A API (RAG + LLM + MongoDB)

A backend system that answers user questions using Retrieval-Augmented Generation (RAG) with a MongoDB knowledge base and an LLM (via Groq/OpenAI).

Built with Node.js, Express, LangChain-style pipeline, JWT auth, and rate limiting.

--------------------------------------------------

Features

- Retrieval-based Q&A (RAG pipeline)
- LLM-powered answer generation
- MongoDB document knowledge base
- JWT Authentication (Register/Login)
- Per-user rate limiting (10 req/min)
- Structured JSON responses (Zod validated)
- Request logging (latency, confidence)
- Jest + Supertest test coverage
- Safe fallback if LLM fails

Setup Instructions

1. Clone Repo

git clone <your-repo-url>
cd smart-qa-api

2. Install Dependencies

npm install

3. Create .env File

PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/smartqa
JWT_SECRET=your_secret_key
GROQ_API_KEY=your_groq_key

4. Seed Database

npm run seed

5. Start Server

npm run dev

6. Run Tests

npm test

Authentication APIs

Register

-H "Content-Type: application/json" \
-d '{

Architecture Overview (RAG Pipeline)

1. User sends question
2. Retrieve relevant documents from MongoDB
3. Inject context into prompt
4. Send to LLM
5. Validate response using Zod
6. Return structured JSON

--------------------------------------------------

Confidence Logic

Confidence is NOT generated by LLM.

Docs Found → Confidence
0 → low
1 → medium
2+ → high

--------------------------------------------------

Security and Safety

- JWT-based authentication
- Rate limiting (10 requests/min per user)
- No hardcoded secrets
- LLM restricted to provided context only
- Safe fallback if LLM fails
